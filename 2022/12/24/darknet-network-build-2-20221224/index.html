code 2.4中代码需要计算很多关于内存指针的偏移（因为物理内存上这些高阶张量都是线性排列的，具体高阶索引体现在特定的指针偏移上），比如#14和#16行都有(i*l.groups + j)这个偏移项，其实就是在考虑组别groups和不同批次batch对内存索引的影响，因为输入输出的特征图大小不同，因此后面接的系数当然也不同，分别是*n*m和*l.c/l.groups*l.h*l.w。
